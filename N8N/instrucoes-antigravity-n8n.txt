====================================
INSTRUÇÕES PARA USO DO MCP N8N COM ANTIGRAVITY
====================================

Este arquivo contém instruções específicas para utilizar o MCP n8n com o Antigravity, permitindo a criação e gerenciamento de fluxos de trabalho no n8n diretamente através do agente de IA.

----------------------------------------
CONFIGURAÇÃO INICIAL
----------------------------------------

1. CARREGAR A SKILL MCP-TOOLS-EXPERT:
   - Inicie uma nova conversa com o Antigravity
   - Digite: /carregar-skill mcp-tools-expert
   - Aguarde a confirmação de que a skill foi carregada

2. CONFIGURAR A FERRAMENTA MCP:
   - Envie a seguinte mensagem:
     "Quero configurar uma nova ferramenta MCP para n8n com os seguintes parâmetros:
     - URL: https://auto.mantovan.com.br/webhook/mcp-tool
     - Método: POST
     - Autenticação: Basic Auth
     - Usuário: mcp_user
     - Senha: [SUBSTITUIR_PELA_SUA_SENHA]
     - Nome da ferramenta: n8n Workflow Creator"

3. REGISTRAR O SCHEMA JSON:
   - Envie: "Por favor, registre este schema JSON para a ferramenta MCP n8n Workflow Creator:"
   - Em seguida, copie e cole o conteúdo do arquivo n8n-mcp-tool-schema.json

4. TESTAR A CONEXÃO:
   - Envie: "Teste a conexão com a ferramenta MCP n8n Workflow Creator usando a ação search_nodes com a query 'webhook'"
   - Verifique se recebe uma resposta válida com resultados da busca

----------------------------------------
COMANDOS BÁSICOS
----------------------------------------

1. BUSCAR NÓS:
   "Busque nós do n8n relacionados a [termo]"
   Exemplo: "Busque nós do n8n relacionados a email"

2. OBTER DETALHES DE UM NÓ:
   "Mostre detalhes do nó [tipo de nó] do n8n"
   Exemplo: "Mostre detalhes do nó n8n-nodes-base.gmail do n8n"

3. BUSCAR TEMPLATES:
   "Busque templates de fluxos de trabalho no n8n para [tarefa]"
   Exemplo: "Busque templates de fluxos de trabalho no n8n para integração com Slack"

4. VALIDAR CONFIGURAÇÃO:
   "Valide esta configuração para o nó [tipo de nó] no n8n:
   {configuração em JSON}"

5. CRIAR WORKFLOW:
   "Crie um fluxo de trabalho no n8n que [descrição da tarefa]"
   Exemplo: "Crie um fluxo de trabalho no n8n que monitore emails e envie notificações"

----------------------------------------
EXEMPLOS PRÁTICOS
----------------------------------------

1. WORKFLOW DE MONITORAMENTO RSS:
   "Crie um workflow no n8n para monitorar um feed RSS e enviar novos itens para o Telegram com as seguintes configurações:
   - URL do feed RSS: https://blog.exemplo.com/rss
   - ID do chat do Telegram: 123456789
   - Incluir título, link e resumo na mensagem
   - Verificar a cada 15 minutos
   - Adicionar tratamento de erros"

2. WORKFLOW DE EMAIL PARA PLANILHA:
   "Desenvolva um workflow no n8n que:
   1. Monitore novos emails no Gmail com um anexo CSV
   2. Extraia os dados do CSV
   3. Armazene esses dados em uma planilha Google
   4. Envie confirmação por email ao remetente
   5. Adicione tratamento de erros em cada etapa"

3. WORKFLOW COM APROVAÇÃO HUMANA:
   "Crie um workflow no n8n para um processo de aprovação onde:
   1. Um formulário é preenchido (use webhook para simular)
   2. Um email é enviado ao gerente para aprovação
   3. O gerente aprova ou rejeita através de botões no email
   4. O resultado é armazenado em banco de dados e notificado ao solicitante"

4. WORKFLOW DE INTEGRAÇÃO DE API:
   "Desenvolva um workflow no n8n que:
   1. Consulte a API do OpenWeatherMap para obter previsão do tempo
   2. Filtre os dados para mostrar apenas temperatura e condições
   3. Formate uma mensagem com essas informações
   4. Envie para um webhook do Discord diariamente às 8h
   5. Armazene histórico em uma planilha para análise de tendências"

----------------------------------------
DICAS PARA RESULTADOS MELHORES
----------------------------------------

1. SEJA ESPECÍFICO:
   - Forneça detalhes claros sobre o que o workflow deve fazer
   - Especifique os nós que deseja utilizar (se souber)
   - Mencione requisitos de autenticação, filtros e formatação

2. CONSTRUA GRADUALMENTE:
   - Comece com workflows simples e adicione complexidade aos poucos
   - Valide cada parte antes de seguir para a próxima
   - Peça ajuda para entender funcionalidades específicas

3. SOLICITE VALIDAÇÃO:
   - Peça ao Antigravity para validar as configurações propostas
   - Solicite explicações sobre as escolhas feitas
   - Valide o workflow completo antes de implementar

4. USE TEMPLATES COMO BASE:
   - Peça para buscar templates relacionados à sua necessidade
   - Solicite adaptações específicas para seu caso de uso
   - Compare diferentes templates para escolher o melhor

5. SOLICITE DOCUMENTAÇÃO:
   - Peça ao Antigravity para adicionar nós de notas com explicações
   - Solicite documentação sobre como o workflow funciona
   - Peça dicas sobre manutenção e possíveis melhorias

----------------------------------------
SOLUÇÃO DE PROBLEMAS COMUNS
----------------------------------------

1. ERRO DE CONEXÃO:
   - Verifique se o workflow MCP está ativo no n8n
   - Confirme se a URL está correta (https://auto.mantovan.com.br/webhook/mcp-tool)
   - Teste o webhook diretamente com uma ferramenta como Postman

2. ERRO DE AUTENTICAÇÃO:
   - Verifique se as credenciais básicas estão corretas
   - Confirme se as credenciais foram enviadas corretamente
   - Temporariamente, teste sem autenticação para isolar o problema

3. ERRO NO SCHEMA:
   - Verifique se o schema JSON foi enviado corretamente e é válido
   - Tente registrar o schema novamente
   - Verifique se não há caracteres especiais ou formatação incorreta

4. ERROS EM AÇÕES ESPECÍFICAS:
   - Verifique a sintaxe dos parâmetros enviados
   - Confirme os parâmetros necessários na documentação
   - Tente uma versão simplificada para isolar o problema

5. ANTIGRAVITY NÃO USA A FERRAMENTA:
   - Certifique-se de que a skill mcp-tools-expert foi carregada
   - Seja explícito na solicitação para usar a ferramenta MCP
   - Reformule seu pedido com mais detalhes sobre a tarefa

----------------------------------------
REFERÊNCIA RÁPIDA DE NÓS POPULARES
----------------------------------------

Aqui estão alguns dos nós mais comuns do n8n que você pode utilizar:

1. TRIGGERS (GATILHOS):
   - n8n-nodes-base.manualTrigger - Execução manual
   - n8n-nodes-base.scheduleTrigger - Agendamento
   - n8n-nodes-base.webhook - Webhook para APIs
   - n8n-nodes-base.gmail - Monitoramento de emails
   - n8n-nodes-base.httpRequest - Polling de APIs

2. PROCESSAMENTO:
   - n8n-nodes-base.set - Definição de variáveis
   - n8n-nodes-base.function - Código JavaScript personalizado
   - n8n-nodes-base.code - Código JavaScript ou Python
   - n8n-nodes-base.if - Condicionais
   - n8n-nodes-base.switch - Múltiplos caminhos condicionais
   - n8n-nodes-base.splitInBatches - Processamento em lotes

3. APIS E INTEGRAÇÕES:
   - n8n-nodes-base.httpRequest - Requisições HTTP
   - n8n-nodes-base.slack - Mensagens no Slack
   - n8n-nodes-base.telegram - Bot do Telegram
   - n8n-nodes-base.googleSheets - Planilhas Google
   - n8n-nodes-base.googleDrive - Arquivos no Drive
   - n8n-nodes-base.microsoftExcel - Excel

4. AI E DADOS:
   - @n8n/n8n-nodes-langchain.agent - Agentes de IA
   - @n8n/n8n-nodes-langchain.lmChatOpenAi - OpenAI Chat
   - n8n-nodes-base.openAi - OpenAI (legado)

COMO ESTRUTURAR UMA SOLICITAÇÃO COMPLETA
----------------------------------------

Para obter os melhores resultados, estruture suas solicitações assim:

1. OBJETIVO CLARO:
   "Preciso criar um workflow no n8n para [objetivo principal]"

2. DETALHAMENTO DO PROCESSO:
   "O workflow deve:
   1. [Primeira etapa]
   2. [Segunda etapa]
   3. [Terceira etapa]..."

3. REQUISITOS ESPECÍFICOS:
   "Requisitos importantes:
   - [Requisito 1]
   - [Requisito 2]
   - [Requisito 3]..."

4. CONSIDERAÇÕES ADICIONAIS:
   "Considere também:
   - Tratamento de erros
   - Performance
   - Segurança dos dados"

5. EXEMPLO:
   "Um exemplo do formato de dados de entrada é: [exemplo]
   O resultado esperado deve ser: [exemplo]"

----------------------------------------
NOTA IMPORTANTE
----------------------------------------

Lembre-se de que qualquer workflow criado com o MCP deve ser revisado antes de ser usado em produção. O Antigravity gera configurações geralmente corretas, mas é importante verificar detalhes específicos de autenticação, permissões e lógica de negócios antes de implantar em ambientes críticos.
